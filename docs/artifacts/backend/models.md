# Backend Models & Migrations

Read this **before writing any SQLAlchemy column, model method, or Alembic migration.**

The codebase uses **SQLAlchemy 2.x async style** (`Mapped[]` + `mapped_column()`) on **SQLite** via `aiosqlite`. Pydantic v2 schemas mirror the models for transport. Migrations are Alembic-autogenerated.

---

## Where models live

```
backend/app/
├── db/
│   ├── base_class.py        # Base + Timestamped mixin
│   ├── session.py           # async engine, SessionLocal, get_db
│   └── types.py             # GUID, EncryptedString, EncryptedJSON
├── models/
│   ├── db_models/           # SQLAlchemy ORM (one file per aggregate root)
│   │   ├── user.py
│   │   ├── chat.py          # Chat, Message, MessageAttachment, MessageEvent, ChatCheckpoint
│   │   ├── workspace.py
│   │   ├── refresh_token.py
│   │   └── enums.py
│   ├── schemas/             # Pydantic v2 request/response models
│   └── types.py             # Cross-cutting TypedDicts and enums
└── migrations/              # Alembic versioned migrations
    ├── env.py               # custom-type rendering (GUID, EncryptedString)
    └── versions/
```

Model files are **per-aggregate-root** (one file per top-level entity), not per-table. Sub-entities (`Message`, `MessageAttachment`, `ChatCheckpoint`) live with their parent (`chat.py`).

## Field rules

### ✅ Mechanized

- mypy strict on `app/` (CI). All `Mapped[]` annotations must be valid.
- Ruff lint + format (CI + pre-commit).

### ⚠️ Advisory (not lint-enforced — agent must remember)

- **Always pair `default=` with `server_default=`.** `default=` only fires through the ORM; data inserted by Alembic, raw SQL, or admin tools won't have it. Example: `is_active: Mapped[bool] = mapped_column(Boolean, default=True, server_default="true", nullable=False)`.
- **Always set `nullable=` explicitly** (never let it default).
- **Always set max length on `String` fields** (e.g., `String(64)`, `String(320)` for emails). Bare `String` is a smell.
- **Always use `DateTime(timezone=True)`** for datetime columns.
- **Don't add `index=True` on an FK column** if a composite index already starts with that column — it'd be redundant and confuses query planners.
- **Use the `GUID` type from `db/types.py`** for `id` columns and FKs. Backed by a `String(36)` on SQLite; transparent UUID round-trip in Python.

### Custom column types — reuse, don't reinvent

| Need | Use | Where |
|---|---|---|
| UUID PK / FK | `GUID()` | `db/types.py` |
| Encrypted string at rest (e.g., GitHub PAT) | `EncryptedString()` | `db/types.py` |
| Encrypted JSON at rest | `EncryptedJSON()` | `db/types.py` |
| Created/updated timestamps | inherit `Base` | `db/base_class.py` (provides `created_at`, `updated_at`) |

Encryption is Fernet-based, derived from `settings.SECRET_KEY`. Migrations render these custom types via the `render_item()` callback in `migrations/env.py` — that's why Alembic autogenerate produces `EncryptedString()` and not `VARCHAR`.

## Relationships

- Use `relationship()` with explicit `back_populates=`.
- Set `cascade="all, delete-orphan"` on parent-side when the child has no meaning without the parent (e.g., `Message` ↔ `MessageAttachment`).
- For tokens that should die with the user, cascade is essential — see PR #586 (cascade refresh tokens on user delete).

---

## Querying

Read this before writing any `select(...)` or `session.execute(...)`:

- **All sessions are async.** Use `async with SessionLocal() as session:` patterns; never sync sessions.
- **Filter in the database, not in Python.** No `[m for m in messages if m.role == "user"]` after a full-table fetch.
- **Use `.exists()` semantics via `select(...).exists()`** not `len(rows) > 0`.
- **Use `.scalar_one()` / `.scalar_one_or_none()`** instead of `.first()` when expecting a single row — clearer about expectations.
- **Eager-load relationships via `selectinload()`** when you'll touch them in a loop. Async lazy-loading raises `MissingGreenlet`.

## Migrations

### File naming

Alembic autogenerate slug names are fine: `b7fae3df7bca_initial_schema.py`, `05d565131567_add_chat_checkpoints.py`. Don't manually rename.

### Hard rules

- **Generate via** `docker compose exec api alembic revision --autogenerate -m "<message>"`. Don't hand-write from scratch.
- **Manual edits to generated migrations are fine for correctness** (e.g., custom type rendering, batch ops for SQLite ALTER limitations) — just don't fabricate the whole file.
- **Never edit a merged migration.** Add a new one.
- **Run alembic inside the Docker backend container** — not on host (your local Python may not match container deps).
- **No data migrations bundled with schema migrations.** Two files, two purposes. (See PR #574 for a squash precedent — coordinated, intentional.)

### SQLite ALTER limitations

SQLite can't drop columns or modify constraints in place. Alembic's `op.batch_alter_table()` produces the table-recreate dance. If autogenerate doesn't do it for you, wrap manually:

```python
with op.batch_alter_table("chats") as batch:
    batch.add_column(...)
```

### Server-default helpers

`db/migrations_helpers.py` (or similar) exposes `uuid_server_default()` and `now_server_default()` for consistency across migrations. Use them — don't write raw `func.now()` strings.

## Schemas (Pydantic v2)

Mirrored under `models/schemas/`. Conventions:

- `ConfigDict(from_attributes=True)` on response models — enables ORM mapping.
- Use `Field(max_length=...)` on **every** `str` field of request models. Add `min_length=1` if empty is invalid. (See `golden_principles.md` §7.)
- Don't use `f"...{user_input}..."` to interpolate — raw concat or `string.Template`.
- **If a response model field has a default, the matching frontend TypeScript type marks it required.** Pydantic defaults always serialize, so the field is always present.

## Model methods

- Keep them small and pure. Side effects belong in services.
- Factory methods go on the class (`Chat.from_dict(...)`), not in services. (See `golden_principles.md` §9.)
- `__repr__` is helpful for debugging; not strictly required.

---

## Anti-patterns to refuse

- ❌ **Bare `String` (no length).** Pick a max length.
- ❌ **`default=` without `server_default=`** (or vice versa) when both should agree.
- ❌ **`nullable=` omitted.** Be explicit.
- ❌ **`DateTime` without `timezone=True`.** Always TZ-aware.
- ❌ **Plaintext storage of secrets.** Use `EncryptedString` for tokens, OAuth creds, anything sensitive. Custom env vars are stored plaintext today (intentional, low-sensitivity, may upgrade later).
- ❌ **Cross-file private method calls** (`_internal()` from another module). Make it public if it's actually shared. (See `golden_principles.md` §8.)
- ❌ **`# type: ignore` on Mapped annotations.** Fix the type, don't silence it.
- ❌ **Hand-written migrations that bypass autogenerate.** Use autogenerate; edit the output if needed.
- ❌ **Editing merged migrations.** New migration to fix; old one stays.
